Регистрация типа записи «product»

Метод register_content_types() вызывает register_post_type с массивом параметров. Тип записи product имеет:

- labels — массив локализованных подписей (название, единственное число, добавление, редактирование, поиск и т.д.);
- public => true — запись видна в админке и на фронтенде;
- show_in_rest => true — доступна через REST API (требуется для Gutenberg и внешних интеграций);
- menu_icon => 'dashicons-products' — иконка в боковом меню;
- supports — массив поддерживаемых возможностей: заголовок, редактор, миниатюра, отрывок;
- has_archive => true — включает архивную страницу /products/;
- rewrite => ['slug' => 'products'] — ЧПУ-префикс для отдельных записей.

Итоговая структура данных

После регистрации плагин создаёт следующую схему:

Тип записи: product
- Таксономии: product_cat (иерарх.), product_tag (неиерарх.)
- Архив: /products/
- ЧПУ: /products/%slug%/

Тип записи: news
- Таксономии: news_category (иерарх.)
- Архив: /news/
- ЧПУ: /news/%slug%/

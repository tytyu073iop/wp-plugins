## 3.5. Реализация Gutenberg-блоков и inline-шорткодов

Плагин регистрирует три Gutenberg-блока и два простых inline-шорткода. Блоки являются динамическими — их разметка генерируется на сервере через PHP-методы, а `save()` возвращает `null`. Inline-шорткоды (`[my_catalog_price]` и `[my_catalog_stock]`) сохранены для обратной совместимости и используются для вывода отдельных полей внутри произвольного содержимого.

### 3.5.1 Регистрация блоков

Все три блока регистрируются в методе `register_block()` координатора `My_Catalog_Plugin`. Каждый блок использует собственный файл `block.json`, расположенный в отдельной директории:

- `block.json` (корень плагина) → блок `my-catalog/news-carousel`
- `blocks/product-table/block.json` → блок `my-catalog/product-table`
- `blocks/product-filters/block.json` → блок `my-catalog/product-filters`

```php
public function register_block() {
    $block_dir          = MY_CATALOG_PATH;
    $product_filter_dir = MY_CATALOG_PATH . 'blocks/product-filters';
    $product_table_dir  = MY_CATALOG_PATH . 'blocks/product-table';

    if ( file_exists( $block_dir . '/block.json' ) ) {
        register_block_type(
            $block_dir,
            array(
                'render_callback' => array( $this->news_carousel, 'render_block' ),
            )
        );
    }

    if ( file_exists( $product_filter_dir . '/block.json' ) ) {
        register_block_type(
            $product_filter_dir,
            array(
                'render_callback' => array( $this->product_table, 'render_filters_block' ),
            )
        );
    }

    if ( file_exists( $product_table_dir . '/block.json' ) ) {
        register_block_type(
            $product_table_dir,
            array(
                'render_callback' => array( $this->product_table, 'render_table_block' ),
            )
        );
    }
}
```

Такая архитектура позволяет WordPress автоматически загружать `editorScript` и `editorStyle` из `block.json` и связывать атрибуты блока с соответствующим render callback'ом.

### 3.5.2 Блок таблицы товаров `my-catalog/product-table`

Блок `my-catalog/product-table` рендерится методом `render_table_block()` класса `My_Catalog_Product_Table`. Вначале атрибуты читаются из массива, переданного WordPress, и применяются значения по умолчанию:

```php
public function render_table_block( $attributes ) {
    $limit         = isset( $attributes['limit'] ) ? $attributes['limit'] : 10;
    $category      = isset( $attributes['category'] ) ? $attributes['category'] : '';
    $tag           = isset( $attributes['tag'] ) ? $attributes['tag'] : '';
    $columns_attr  = isset( $attributes['columns'] ) ? $attributes['columns'] : '';
    $search        = ! empty( $attributes['search'] );
    $empty_message = isset( $attributes['emptyMessage'] )
        ? $attributes['emptyMessage']
        : __( 'No products found.', 'my-catalog' );
    $table_id      = isset( $attributes['tableId'] ) ? $attributes['tableId'] : '';
    // ...
}
```

Далее определяется набор колонок. Если атрибут `columns` не указан (пустая строка), используются колонки, сохранённые в настройках плагина. В противном случае переданный список разделяется по запятой, каждый ключ санитаризуется через `sanitize_key()` и пересекается с массивом доступных колонок, отбрасывая невалидные значения:

```php
$column_keys = $this->parse_columns_attribute( $columns_attr );

private function parse_columns_attribute( $columns ) {
    if ( empty( $columns ) ) {
        return $this->get_selected_columns();
    }

    return $this->sanitize_columns(
        array_map( 'trim', explode( ',', (string) $columns ) )
    );
}
```

Метод `get_selected_columns()` читает сохранённую опцию `my_catalog_product_table_columns`, а при её отсутствии возвращает колонки по умолчанию: `image`, `title`, `price`, `category`, `sku`.

#### Генерация HTML-разметки

После парсинга атрибутов метод `render_table_block()` формирует JSON-конфигурацию, которая записывается в data-атрибут контейнера. Эта конфигурация содержит все параметры, необходимые JavaScript для инициализации DataTables:

```php
$columns     = $this->prepare_columns_for_frontend( $column_keys );
$instance_id = sanitize_html_class( $table_id )
    ?: 'my-catalog-product-table-' . ++self::$instance;

$config = array(
    'restUrl'      => rest_url( 'my-catalog/v1/product-table' ),
    'baseCategory' => sanitize_text_field( $category ),
    'baseTag'      => sanitize_text_field( $tag ),
    'columns'      => $columns,
    'pageLength'   => max( 1, absint( $limit ) ),
    'search'       => $search,
    'defaultOrder' => $this->get_default_order_for_columns( $columns ),
);
```

Каждая колонка в конфигурации представлена массивом с ключами `key`, `label` и `orderable`. Метод `get_default_order_for_columns()` определяет первую сортируемую колонку и задаёт порядок сортировки по умолчанию (например, `[1, 'asc']` для колонки «Name»), что позволяет DataTables отображать отсортированные данные с первого запроса.

Генерация HTML-разметки использует буферизацию вывода (`ob_start()` / `ob_get_clean()`). Таблица рендерится с пустым телом — единственная строка содержит сообщение об отсутствии товаров, переданное через атрибут `emptyMessage`. Реальные данные загружаются асинхронно после инициализации DataTables на стороне клиента:

```php
ob_start();
?>
<div class="my-catalog-product-table" id="<?php echo esc_attr( $instance_id ); ?>"
     data-config="<?php echo esc_attr( wp_json_encode( $config ) ); ?>">
    <div class="my-catalog-product-table__frame">
        <table class="display responsive nowrap" style="width:100%">
            <thead>
                <tr>
                    <?php foreach ( $columns as $column ) : ?>
                        <th><?php echo esc_html( $column['label'] ); ?></th>
                    <?php endforeach; ?>
                </tr>
            </thead>
            <tbody>
                <tr>
                    <td colspan="<?php echo esc_attr( count( $columns ) ); ?>">
                        <?php echo esc_html( $empty_message ); ?>
                    </td>
                </tr>
            </tbody>
        </table>
    </div>
</div>
<?php
return (string) ob_get_clean();
```

Перед рендерингом подключаются стили и скрипты, зарегистрированные через `wp_register_style()` / `wp_register_script()`. DataTables 1.13.8 загружается с CDN; JavaScript-файл плагина зависит от jQuery:

```php
wp_enqueue_style( 'my-catalog-product-table' );
wp_enqueue_script( 'my-catalog-product-table' );
```

JavaScript-код в `assets/js/product-table.js` после загрузки страницы находит все контейнеры с классом `my-catalog-product-table`, читает их `data-config` и инициализирует DataTables в серверном режиме (`serverSide: true`). Конфигурация `ajax.data()` формирует параметры запроса, передавая REST endpoint'у draw, пагинацию, поиск, сортировку, колонки и базовые фильтры:

```javascript
$(function () {
    $('.my-catalog-product-table').each(function () {
        var wrapper = this;
        var config = JSON.parse(wrapper.getAttribute('data-config'));

        var columns = (config.columns || []).map(function (column) {
            return {
                data: column.key,
                name: column.key,
                orderable: !!column.orderable,
                searchable: true,
                defaultContent: '\u2014',
            };
        });

        var table = $(wrapper).find('table').DataTable({
            processing: true,
            serverSide: true,
            searching: config.search !== false,
            pageLength: config.pageLength || 10,
            responsive: true,
            autoWidth: false,
            order: [config.defaultOrder || [0, 'asc']],
            ajax: {
                url: config.restUrl,
                type: 'GET',
                data: function (data) {
                    var order = data.order && data.order[0]
                        ? data.order[0]
                        : { column: 0, dir: 'asc' };
                    var orderColumn = columns[order.column]
                        ? columns[order.column].name
                        : 'title';
                    return {
                        draw: data.draw,
                        start: data.start,
                        length: data.length,
                        search: data.search ? data.search.value : '',
                        order_column: orderColumn,
                        order_dir: order.dir || 'asc',
                        category: '',
                        tag: '',
                        price_min: '',
                        price_max: '',
                        base_category: config.baseCategory || '',
                        base_tag: config.baseTag || '',
                        columns: columns.map(function (col) { return col.name; }),
                    };
                },
            },
            columns: columns,
        });
    });
});
```

Ответ REST endpoint'а следует протоколу DataTables: `draw`, `recordsTotal`, `recordsFiltered`, `data`. После получения ответа DataTables автоматически обновляет таблицу с поддержкой пагинации, сортировки и поиска без перезагрузки страницы.

### 3.5.3 Блок фильтров товаров `my-catalog/product-filters`

Блок `my-catalog/product-filters` предоставляет фильтры, которые могут быть размещены отдельно от таблицы. Он рендерит выпадающий список категорий и диапазон цен с двумя бегунками и синхронизированными числовыми полями.

Рендеринг выполняется методом `render_filters_block()`:

```php
public function render_filters_block( $attributes ) {
    $target        = isset( $attributes['target'] ) ? $attributes['target'] : '';
    $show_category = ! empty( $attributes['showCategory'] );
    $show_price    = ! empty( $attributes['showPrice'] );

    wp_enqueue_style( 'my-catalog-product-table' );
    wp_enqueue_script( 'my-catalog-product-table' );

    return $this->render_filter_controls(
        array(
            'target'        => sanitize_html_class( $target ),
            'show_category' => $show_category,
            'show_price'    => $show_price,
            'external'      => true,
        )
    );
}
```

Метод `render_filter_controls()` генерирует разметку фильтров. Для категорий строится выпадающий список `<select>` с опциями, загружаемыми из таксономии `product_cat`. Для цен — двойной ползунок с числовыми полями ввода:

Границы ценового диапазона вычисляются на основе существующих товаров через прямой SQL-запрос, что гарантирует актуальность ползунков даже после добавления или удаления товаров:

```php
private function get_price_filter_bounds() {
    global $wpdb;

    $bounds = $wpdb->get_row(
        $wpdb->prepare(
            "SELECT MIN(CAST(meta_value AS DECIMAL(20,4))) AS min_price,
                    MAX(CAST(meta_value AS DECIMAL(20,4))) AS max_price
             FROM {$wpdb->postmeta}
             WHERE meta_key = %s
               AND meta_value != ''
               AND meta_value REGEXP '^[0-9]+(\\\\.[0-9]+)?$'",
            My_Catalog_Core::PRODUCT_META_PRICE
        ),
        ARRAY_A
    );

    $min = isset( $bounds['min_price'] ) ? floor( (float) $bounds['min_price'] ) : 0;
    $max = isset( $bounds['max_price'] ) ? ceil( (float) $bounds['max_price'] ) : 0;

    if ( $max <= $min ) {
        $max = $min + 1;
    }

    return array( 'min' => $min, 'max' => $max );
}
```

Фильтры связываются с таблицей через атрибут `data-target` — ID контейнера таблицы (значение атрибута `tableId` блока «Product Table»). Если `target` не указан, JavaScript-код применяет фильтры ко всем таблицам на странице:

```javascript
var $externalFilters = $('.my-catalog-product-filters').filter(function () {
    var target = $(this).data('target');
    return !target || target === wrapper.id;
});
```

При изменении любого фильтра вызывается `table.ajax.reload()`, который отправляет новый запрос к REST API с обновлёнными параметрами `category`, `price_min` и `price_max`, а таблица перерисовывается с новыми данными:

```javascript
$category.on('change', function () { table.ajax.reload(); });

$priceRanges.on('input change', 'input', function () {
    syncPriceRange($field.closest('.js-my-catalog-product-price-range'), changedField);
    table.ajax.reload();
});
```

Синхронизация числовых полей с бегунками осуществляется функцией `syncPriceRange()`, которая обновляет значения при любом изменении — будь то ввод в текстовое поле или перемещение ползунка. Визуальный трек закрашивается между двумя бегунками через CSS-свойства `--range-left` и `--range-right`, управляемые функцией `updatePriceRangeTrack()`.

### 3.5.4 Простые inline-шорткоды

Шорткоды `[my_catalog_price]` и `[my_catalog_stock]` предназначены для вывода отдельных мета-полей текущего товара внутри произвольного содержимого. Они не принимают атрибутов и определяют контекст через `get_the_ID()` внутри цикла WordPress.

`[my_catalog_price]` выводит цену товара, отформатированную через `number_format_i18n()` (с учётом локали сайта). Если текущая запись не является товаром или цена не указана, шорткод возвращает пустую строку:

```php
public function render_price_shortcode() {
    $post_id = get_the_ID();

    if ( ! $post_id
        || My_Catalog_Core::PRODUCT_POST_TYPE !== get_post_type( $post_id )
    ) {
        return '';
    }

    $price = get_post_meta(
        $post_id, My_Catalog_Core::PRODUCT_META_PRICE, true
    );

    if ( '' === $price ) {
        return '';
    }

    return esc_html( number_format_i18n( (float) $price, 2 ) );
}
```

`[my_catalog_stock]` выводит локализованную метку статуса наличия. Статус читается из мета-поля `_my_catalog_stock_status` и маппится через `get_stock_statuses()`:

```php
public function render_stock_shortcode() {
    $post_id = get_the_ID();

    if ( ! $post_id
        || My_Catalog_Core::PRODUCT_POST_TYPE !== get_post_type( $post_id )
    ) {
        return '';
    }

    $stock_status = get_post_meta(
        $post_id, My_Catalog_Core::PRODUCT_META_STOCK, true
    );
    $stock_map    = My_Catalog_Core::get_stock_statuses();

    if ( ! isset( $stock_map[ $stock_status ] ) ) {
        return '';
    }

    return esc_html( $stock_map[ $stock_status ] );
}
```

Оба шорткода возвращают отформатированную строку (экранирование происходит внутри метода через `esc_html()`), что позволяет использовать их в любом контексте: в теле записи, в виджетах или в произвольных шаблонах. Пример использования внутри цикла:

```
Цена: [my_catalog_price] руб. — Статус: [my_catalog_stock]
```

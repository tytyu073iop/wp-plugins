## 3.5. Реализация шорткодов

Плагин регистрирует четыре шорткода, все в конструкторе класса `My_Catalog_Product_Table`:

```php
add_shortcode( 'product_table',     array( $this, 'render_shortcode' ) );
add_shortcode( 'product_filters',   array( $this, 'render_filters_shortcode' ) );
add_shortcode( 'my_catalog_price',  array( $this, 'render_price_shortcode' ) );
add_shortcode( 'my_catalog_stock',  array( $this, 'render_stock_shortcode' ) );
```

Два первых (`[product_table]`, `[product_filters]`) являются составными — генерируют сложную HTML-разметку с JavaScript-инициализацией DataTables и взаимодействием через REST API. Два последних (`[my_catalog_price]`, `[my_catalog_stock]`) — простые inline-шорткоды для вывода отдельных полей текущего товара.

### 3.5.1 Парсинг атрибутов `[product_table]`

Шорткод `[product_table]` принимает следующие атрибуты:

- `limit` — количество записей на странице (по умолчанию 10);
- `category` — фильтрация по категории товара (slug или ID);
- `tag` — фильтрация по атрибуту/тегу;
- `columns` — список отображаемых колонок через запятую (переопределяет настройки);
- `search` — включение (`true`) или отключение (`false`) строки поиска;
- `table_id` — кастомный идентификатор для связи с фильтрами;
- `empty_message` — сообщение при отсутствии товаров.

Обработка атрибутов выполняется в методе `render_shortcode()` класса `My_Catalog_Product_Table`. Вначале атрибуты сливаются со значениями по умолчанию через `shortcode_atts()`:

```php
public function render_shortcode( $atts ) {
    $atts = shortcode_atts(
        array(
            'limit'         => 10,
            'category'      => '',
            'tag'           => '',
            'columns'       => '',
            'search'        => 'true',
            'empty_message' => __( 'No products found.', 'my-catalog' ),
            'table_id'      => '',
        ),
        $atts,
        'product_table'
    );
    // ...
}
```

Далее определяется набор колонок. Если атрибут `columns` не указан (пустая строка), используются колонки, сохранённые в настройках плагина. В противном случае переданный список разделяется по запятой, каждый ключ санитаризуется через `sanitize_key()` и пересекается с массивом доступных колонок, отбрасывая невалидные значения:

```php
$column_keys = $this->parse_columns_attribute( $atts['columns'] );

private function parse_columns_attribute( $columns_attr ) {
    if ( empty( $columns_attr ) ) {
        return $this->get_selected_columns();
    }

    $columns   = array_map( 'sanitize_key', explode( ',', $columns_attr ) );
    $available = array_keys( $this->get_available_columns() );
    $columns   = array_values( array_intersect( $columns, $available ) );

    return ! empty( $columns ) ? $columns : $this->get_selected_columns();
}
```

Метод `get_selected_columns()` читает сохранённую опцию `my_catalog_product_table_columns`, а при её отсутствии возвращает колонки по умолчанию: `image`, `title`, `price`, `category`, `sku`.

### 3.5.2 Генерация HTML-разметки

После парсинга атрибутов метод `render_shortcode()` формирует JSON-конфигурацию, которая записывается в data-атрибут контейнера. Эта конфигурация содержит все параметры, необходимые JavaScript для инициализации DataTables:

```php
$columns     = $this->prepare_columns_for_frontend( $column_keys );
$instance_id = sanitize_html_class( $atts['table_id'] )
    ?: 'my-catalog-product-table-' . ++self::$instance;

$config = array(
    'restUrl'      => rest_url( 'my-catalog/v1/product-table' ),
    'baseCategory' => sanitize_text_field( $atts['category'] ),
    'baseTag'      => sanitize_text_field( $atts['tag'] ),
    'columns'      => $columns,
    'pageLength'   => max( 1, absint( $atts['limit'] ) ),
    'search'       => filter_var( $atts['search'], FILTER_VALIDATE_BOOLEAN ),
    'defaultOrder' => $this->get_default_order_for_columns( $columns ),
);
```

Каждая колонка в конфигурации представлена массивом с ключами `key`, `label` и `orderable`. Метод `get_default_order_for_columns()` определяет первую сортируемую колонку и задаёт порядок сортировки по умолчанию (например, `[1, 'asc']` для колонки «Name»), что позволяет DataTables отображать отсортированные данные с первого запроса.

Генерация HTML-разметки использует буферизацию вывода (`ob_start()` / `ob_get_clean()`). Таблица рендерится с пустым телом — единственная строка содержит сообщение об отсутствии товаров, переданное через атрибут `empty_message`. Реальные данные загружаются асинхронно после инициализации DataTables на стороне клиента:

```php
ob_start();
?>
<div class="my-catalog-product-table" id="<?php echo esc_attr( $instance_id ); ?>"
     data-config="<?php echo esc_attr( wp_json_encode( $config ) ); ?>">
    <div class="my-catalog-product-table__frame">
        <table class="display responsive nowrap" style="width:100%">
            <thead>
                <tr>
                    <?php foreach ( $columns as $column ) : ?>
                        <th><?php echo esc_html( $column['label'] ); ?></th>
                    <?php endforeach; ?>
                </tr>
            </thead>
            <tbody>
                <tr>
                    <td colspan="<?php echo esc_attr( count( $columns ) ); ?>">
                        <?php echo esc_html( $atts['empty_message'] ); ?>
                    </td>
                </tr>
            </tbody>
        </table>
    </div>
</div>
<?php
return (string) ob_get_clean();
```

Перед рендерингом подключаются стили и скрипты, зарегистрированные через `wp_register_style()` / `wp_register_script()`. DataTables 1.13.8 загружается с CDN; JavaScript-файл плагина зависит от jQuery:

```php
wp_enqueue_style( 'my-catalog-product-table' );
wp_enqueue_script( 'my-catalog-product-table' );
```

JavaScript-код в `assets/js/product-table.js` после загрузки страницы находит все контейнеры с классом `my-catalog-product-table`, читает их `data-config` и инициализирует DataTables в серверном режиме (`serverSide: true`). Конфигурация `ajax.data()` формирует параметры запроса, передавая REST endpoint'у draw, пагинацию, поиск, сортировку, колонки и базовые фильтры:

```javascript
$(function () {
    $('.my-catalog-product-table').each(function () {
        var wrapper = this;
        var config = JSON.parse(wrapper.getAttribute('data-config'));

        var columns = (config.columns || []).map(function (column) {
            return {
                data: column.key,
                name: column.key,
                orderable: !!column.orderable,
                searchable: true,
                defaultContent: '\u2014',
            };
        });

        var table = $(wrapper).find('table').DataTable({
            processing: true,
            serverSide: true,
            searching: config.search !== false,
            pageLength: config.pageLength || 10,
            responsive: true,
            autoWidth: false,
            order: [config.defaultOrder || [0, 'asc']],
            ajax: {
                url: config.restUrl,
                type: 'GET',
                data: function (data) {
                    var order = data.order && data.order[0]
                        ? data.order[0]
                        : { column: 0, dir: 'asc' };
                    var orderColumn = columns[order.column]
                        ? columns[order.column].name
                        : 'title';
                    return {
                        draw: data.draw,
                        start: data.start,
                        length: data.length,
                        search: data.search ? data.search.value : '',
                        order_column: orderColumn,
                        order_dir: order.dir || 'asc',
                        category: '',
                        tag: '',
                        price_min: '',
                        price_max: '',
                        base_category: config.baseCategory || '',
                        base_tag: config.baseTag || '',
                        columns: columns.map(function (col) { return col.name; }),
                    };
                },
            },
            columns: columns,
        });
    });
});
```

Ответ REST endpoint'а следует протоколу DataTables: `draw`, `recordsTotal`, `recordsFiltered`, `data`. После получения ответа DataTables автоматически обновляет таблицу с поддержкой пагинации, сортировки и поиска без перезагрузки страницы.

### 3.5.3 Фильтры товаров

Шорткод `[product_filters]` предоставляет блок фильтров, который может быть размещён отдельно от таблицы. Он рендерит выпадающий список категорий и диапазон цен с двумя бегунками и синхронизированными числовыми полями.

Парсинг атрибутов выполняется методом `render_filters_shortcode()`:

```php
public function render_filters_shortcode( $atts ) {
    $atts = shortcode_atts(
        array(
            'target'        => '',
            'show_category' => true,
            'show_price'    => true,
        ),
        $atts,
        'product_filters'
    );

    return $this->render_filter_controls( array(
        'target'        => sanitize_html_class( $atts['target'] ),
        'show_category' => filter_var( $atts['show_category'], FILTER_VALIDATE_BOOLEAN ),
        'show_price'    => filter_var( $atts['show_price'], FILTER_VALIDATE_BOOLEAN ),
        'external'      => true,
    ) );
}
```

Метод `render_filter_controls()` генерирует разметку фильтров. Для категорий строится выпадающий список `<select>` с опциями, загружаемыми из таксономии `product_cat`. Для цен — двойной ползунок с числовыми полями ввода:

```php
private function render_filter_controls( $args = array() ) {
    $args = wp_parse_args( $args, array(
        'target'        => '',
        'show_category' => true,
        'show_price'    => true,
        'external'      => false,
    ) );

    $price_bounds = $this->get_price_filter_bounds();

    ob_start();
    ?>
    <div class="my-catalog-product-table__filters<?php
        echo ! empty( $args['external'] )
            ? ' my-catalog-product-filters' : '';
    ?>" <?php echo ! empty( $args['target'] )
        ? 'data-target="' . esc_attr( $args['target'] ) . '"'
        : ''; ?>>

        <?php if ( ! empty( $args['show_category'] ) ) : ?>
            <label class="my-catalog-product-table__filter">
                <span><?php esc_html_e( 'Category', 'my-catalog' ); ?></span>
                <select class="js-my-catalog-product-category">
                    <option value="">
                        <?php esc_html_e( 'All categories', 'my-catalog' ); ?>
                    </option>
                    <?php foreach ( $this->get_terms_for_filter(
                        My_Catalog_Core::PRODUCT_CATEGORY_TAXONOMY
                    ) as $term ) : ?>
                        <option value="<?php echo esc_attr( $term->slug ); ?>">
                            <?php echo esc_html( $term->name ); ?>
                        </option>
                    <?php endforeach; ?>
                </select>
            </label>
        <?php endif; ?>

        <?php if ( ! empty( $args['show_price'] ) ) : ?>
            <div class="my-catalog-product-table__filter
                        my-catalog-product-table__filter--price-range
                        js-my-catalog-product-price-range"
                 data-min="<?php echo esc_attr( $price_bounds['min'] ); ?>"
                 data-max="<?php echo esc_attr( $price_bounds['max'] ); ?>"
                 data-step="0.01">
                <span><?php esc_html_e( 'Price', 'my-catalog' ); ?></span>
                <div class="my-catalog-product-table__price-fields">
                    <label>
                        <span><?php esc_html_e( 'Min', 'my-catalog' ); ?></span>
                        <input class="js-my-catalog-product-price-min" type="number"
                               min="<?php echo esc_attr( $price_bounds['min'] ); ?>"
                               max="<?php echo esc_attr( $price_bounds['max'] ); ?>"
                               step="0.01"
                               value="<?php echo esc_attr( $price_bounds['min'] ); ?>" />
                    </label>
                    <label>
                        <span><?php esc_html_e( 'Max', 'my-catalog' ); ?></span>
                        <input class="js-my-catalog-product-price-max" type="number"
                               min="<?php echo esc_attr( $price_bounds['min'] ); ?>"
                               max="<?php echo esc_attr( $price_bounds['max'] ); ?>"
                               step="0.01"
                               value="<?php echo esc_attr( $price_bounds['max'] ); ?>" />
                    </label>
                </div>
                <div class="my-catalog-product-table__range-slider">
                    <div class="my-catalog-product-table__range-track"></div>
                    <input class="js-my-catalog-product-price-min-range" type="range"
                           min="<?php echo esc_attr( $price_bounds['min'] ); ?>"
                           max="<?php echo esc_attr( $price_bounds['max'] ); ?>"
                           step="0.01"
                           value="<?php echo esc_attr( $price_bounds['min'] ); ?>" />
                    <input class="js-my-catalog-product-price-max-range" type="range"
                           min="<?php echo esc_attr( $price_bounds['min'] ); ?>"
                           max="<?php echo esc_attr( $price_bounds['max'] ); ?>"
                           step="0.01"
                           value="<?php echo esc_attr( $price_bounds['max'] ); ?>" />
                </div>
            </div>
        <?php endif; ?>
    </div>
    <?php
    return (string) ob_get_clean();
}
```

Границы ценового диапазона вычисляются на основе существующих товаров через прямой SQL-запрос, что гарантирует актуальность ползунков даже после добавления или удаления товаров:

```php
private function get_price_filter_bounds() {
    global $wpdb;

    $bounds = $wpdb->get_row(
        $wpdb->prepare(
            "SELECT MIN(CAST(meta_value AS DECIMAL(20,4))) AS min_price,
                    MAX(CAST(meta_value AS DECIMAL(20,4))) AS max_price
             FROM {$wpdb->postmeta}
             WHERE meta_key = %s
               AND meta_value != ''
               AND meta_value REGEXP '^[0-9]+(\\\\.[0-9]+)?$'",
            My_Catalog_Core::PRODUCT_META_PRICE
        ),
        ARRAY_A
    );

    $min = isset( $bounds['min_price'] ) ? floor( (float) $bounds['min_price'] ) : 0;
    $max = isset( $bounds['max_price'] ) ? ceil( (float) $bounds['max_price'] ) : 0;

    if ( $max <= $min ) {
        $max = $min + 1;
    }

    return array( 'min' => $min, 'max' => $max );
}
```

Фильтры связываются с таблицей через атрибут `data-target` — ID контейнера таблицы (значение атрибута `table_id` шорткода `[product_table]`). Если `target` не указан, JavaScript-код применяет фильтры ко всем таблицам на странице:

```javascript
var $externalFilters = $('.my-catalog-product-filters').filter(function () {
    var target = $(this).data('target');
    return !target || target === wrapper.id;
});
```

При изменении любого фильтра вызывается `table.ajax.reload()`, который отправляет новый запрос к REST API с обновлёнными параметрами `category`, `price_min` и `price_max`, а таблица перерисовывается с новыми данными:

```javascript
$category.on('change', function () { table.ajax.reload(); });

$priceRanges.on('input change', 'input', function () {
    syncPriceRange($field.closest('.js-my-catalog-product-price-range'), changedField);
    table.ajax.reload();
});
```

Синхронизация числовых полей с бегунками осуществляется функцией `syncPriceRange()`, которая обновляет значения при любом изменении — будь то ввод в текстовое поле или перемещение ползунка. Визуальный трек закрашивается между двумя бегунками через CSS-свойства `--range-left` и `--range-right`, управляемые функцией `updatePriceRangeTrack()`.

### 3.5.4 Простые inline-шорткоды

Шорткоды `[my_catalog_price]` и `[my_catalog_stock]` предназначены для вывода отдельных мета-полей текущего товара внутри произвольного содержимого. Они не принимают атрибутов и определяют контекст через `get_the_ID()` внутри цикла WordPress.

`[my_catalog_price]` выводит цену товара, отформатированную через `number_format_i18n()` (с учётом локали сайта). Если текущая запись не является товаром или цена не указана, шорткод возвращает пустую строку:

```php
public function render_price_shortcode() {
    $post_id = get_the_ID();

    if ( ! $post_id
        || My_Catalog_Core::PRODUCT_POST_TYPE !== get_post_type( $post_id )
    ) {
        return '';
    }

    $price = get_post_meta(
        $post_id, My_Catalog_Core::PRODUCT_META_PRICE, true
    );

    if ( '' === $price ) {
        return '';
    }

    return esc_html( number_format_i18n( (float) $price, 2 ) );
}
```

`[my_catalog_stock]` выводит локализованную метку статуса наличия. Статус читается из мета-поля `_my_catalog_stock_status` и маппится через `get_stock_statuses()`:

```php
public function render_stock_shortcode() {
    $post_id = get_the_ID();

    if ( ! $post_id
        || My_Catalog_Core::PRODUCT_POST_TYPE !== get_post_type( $post_id )
    ) {
        return '';
    }

    $stock_status = get_post_meta(
        $post_id, My_Catalog_Core::PRODUCT_META_STOCK, true
    );
    $stock_map    = My_Catalog_Core::get_stock_statuses();

    if ( ! isset( $stock_map[ $stock_status ] ) ) {
        return '';
    }

    return esc_html( $stock_map[ $stock_status ] );
}
```

Оба шорткода возвращают отформатированную строку (не экранируют на выводе — экранирование происходит внутри метода через `esc_html()`), что позволяет использовать их в любом контексте: в теле записи, в виджетах или в произвольных шаблонах. Пример использования внутри цикла:

```
Цена: [my_catalog_price] руб. — Статус: [my_catalog_stock]
```

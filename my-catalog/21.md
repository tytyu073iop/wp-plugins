### 2.1. Регистрация произвольных типов записей (product, news) и таксономий

Первый шаг при разработке плагина — создание собственных типов записей (Custom Post Types) и таксономий для их классификации. В плагине «My Catalog» эта задача решается в классе `My_Catalog_Core` (файл `includes/class-my-catalog-core.php`). Конструктор класса подписывает метод `register_content_types` на событие `init`:

```php
public function __construct() {
    add_action( 'init', array( __CLASS__, 'register_content_types' ) );
}
```

Для каждого типа записи и таксономии заведена именованная константа — это исключает ошибки в написании строк и упрощает рефакторинг:

```php
const PRODUCT_POST_TYPE       = 'product';
const PRODUCT_CATEGORY_TAXONOMY = 'product_cat';
const PRODUCT_TAG_TAXONOMY    = 'product_tag';
const NEWS_POST_TYPE          = 'news';
const NEWS_CATEGORY_TAXONOMY  = 'news_category';
```

#### Регистрация типа записи «product»

Метод `register_content_types()` вызывает `register_post_type` с массивом параметров. Тип записи `product` имеет:

- **labels** — массив локализованных подписей (название, единственное число, добавление, редактирование, поиск и т.д.);
- **public => true** — запись видна в админке и на фронтенде;
- **show_in_rest => true** — доступна через REST API (требуется для Gutenberg и внешних интеграций);
- **menu_icon => 'dashicons-products'** — иконка в боковом меню;
- **supports** — массив поддерживаемых возможностей: заголовок, редактор, миниатюра, отрывок;
- **has_archive => true** — включает архивную страницу `/products/`;
- **rewrite => ['slug' => 'products']** — ЧПУ-префикс для отдельных записей.

```php
register_post_type(
    self::PRODUCT_POST_TYPE,
    array(
        'labels'       => array(
            'name'               => __( 'Products', 'my-catalog' ),
            'singular_name'      => __( 'Product', 'my-catalog' ),
            'add_new_item'       => __( 'Add Product', 'my-catalog' ),
            'edit_item'          => __( 'Edit Product', 'my-catalog' ),
            'new_item'           => __( 'New Product', 'my-catalog' ),
            'view_item'          => __( 'View Product', 'my-catalog' ),
            'search_items'       => __( 'Search Products', 'my-catalog' ),
            'not_found'          => __( 'No products found.', 'my-catalog' ),
            'not_found_in_trash' => __( 'No products found in Trash.', 'my-catalog' ),
            'menu_name'          => __( 'Products', 'my-catalog' ),
        ),
        'public'       => true,
        'show_in_rest' => true,
        'menu_icon'    => 'dashicons-products',
        'supports'     => array( 'title', 'editor', 'thumbnail', 'excerpt' ),
        'has_archive'  => true,
        'rewrite'      => array( 'slug' => 'products' ),
    )
);
```

#### Регистрация таксономий для товаров

Для группировки товаров по категориям используется иерархическая таксономия `product_cat` (аналог категорий). Для неструктурированных признаков (цвет, размер, материал) — неиерархическая `product_tag`, которая в интерфейсе подписана как «Product Attributes»:

```php
// Иерархическая — категории товаров
register_taxonomy(
    self::PRODUCT_CATEGORY_TAXONOMY,
    self::PRODUCT_POST_TYPE,
    array(
        'labels'            => array(
            'name'          => __( 'Product Categories', 'my-catalog' ),
            'singular_name' => __( 'Product Category', 'my-catalog' ),
        ),
        'public'            => true,
        'show_in_rest'      => true,
        'hierarchical'      => true,
        'show_admin_column' => true,
        'rewrite'           => array( 'slug' => 'product-category' ),
    )
);
```

Обе таксономии:
- доступны через REST API (`show_in_rest => true`);
- отображаются в колонке таблицы записей (`show_admin_column => true`).

#### Регистрация типа записи «news»

Тип записи `news` регистрируется по тому же шаблону. Принципиальное отличие — другой набор меток и ЧПУ-префикс `news`. Интерфейсные возможности совпадают с `product`:

```php
register_post_type(
    self::NEWS_POST_TYPE,
    array(
        'labels'       => array(
            'name'               => __( 'News', 'my-catalog' ),
            'singular_name'      => __( 'News Item', 'my-catalog' ),
            'add_new_item'       => __( 'Add News Item', 'my-catalog' ),
            'edit_item'          => __( 'Edit News Item', 'my-catalog' ),
            // ... остальные метки
        ),
        'public'       => true,
        'show_in_rest' => true,
        'menu_icon'    => 'dashicons-megaphone',
        'supports'     => array( 'title', 'editor', 'thumbnail', 'excerpt' ),
        'has_archive'  => true,
        'rewrite'      => array( 'slug' => 'news' ),
    )
);
```

Для новостей определена таксономия `news_category` — иерархическая, с префиксом `news-category`. Это позволяет сортировать новости по тематическим рубрикам.

#### Итоговая структура данных

После регистрации плагин создаёт следующую схему:

| Тип записи | Таксономии | Архив | ЧПУ |
|---|---|---|---|
| `product` | `product_cat` (иерарх.), `product_tag` (неиерарх.) | `/products/` | `/products/%slug%/` |
| `news` | `news_category` (иерарх.) | `/news/` | `/news/%slug%/` |

Для корректной работы ЧПУ-префиксов необходимо сбрасывать правила перезаписи при активации плагина. Этот сброс выполняется в файле `my-catalog.php` через хук активации:

```php
register_activation_hook( __FILE__, function () {
    My_Catalog_Core::register_content_types();
    flush_rewrite_rules();
    update_option( 'my_catalog_version', MY_CATALOG_VERSION );
} );
```

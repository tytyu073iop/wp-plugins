## 3.10. Пример использования

### 3.10.1 Создание товаров автором

После активации плагина в админ-панели WordPress появляется пункт меню «Products» (Рисунок 3.5). Автор может создавать товары, заполняя стандартные поля (заголовок, описание, изображение через миниатюру) и мета-бокс «Product Details», добавленный плагином.

[Место для Рисунка 3.5 — Скриншот экрана создания товара с мета-боксом «Product Details»]

Мета-бокс регистрируется в методе `register_meta_boxes()` класса `My_Catalog_Core` через вызов `add_meta_box()`:

```php
public function register_meta_boxes() {
    add_meta_box(
        'my-catalog-product-details',
        __( 'Product Details', 'my-catalog' ),
        array( $this, 'render_product_meta_box' ),
        self::PRODUCT_POST_TYPE,
        'normal',
        'default'
    );
}
```

Метод `render_product_meta_box()` выводит HTML-форму с пятью полями: цена, артикул, статус наличия, вес, внешняя ссылка магазина. Все поля используют защищённые мета-ключи с префиксом `_my_catalog_`:

```php
public function render_product_meta_box( $post ) {
    wp_nonce_field( 'my_catalog_save_product_meta', 'my_catalog_product_meta_nonce' );

    $price        = get_post_meta( $post->ID, self::PRODUCT_META_PRICE, true );
    $sku          = get_post_meta( $post->ID, self::PRODUCT_META_SKU, true );
    $stock_status = get_post_meta( $post->ID, self::PRODUCT_META_STOCK, true );
    $weight       = get_post_meta( $post->ID, self::PRODUCT_META_WEIGHT, true );
    $external_url = get_post_meta( $post->ID, self::PRODUCT_META_EXTERNAL_URL, true );
    ?>
    <table class="form-table" role="presentation">
        <tbody>
            <tr>
                <th scope="row">
                    <label for="my-catalog-price"><?php esc_html_e( 'Price', 'my-catalog' ); ?></label>
                </th>
                <td>
                    <input type="text" class="regular-text" id="my-catalog-price"
                           name="my_catalog_price"
                           value="<?php echo esc_attr( $price ); ?>" />
                </td>
            </tr>
            <tr>
                <th scope="row">
                    <label for="my-catalog-sku"><?php esc_html_e( 'SKU', 'my-catalog' ); ?></label>
                </th>
                <td>
                    <input type="text" class="regular-text" id="my-catalog-sku"
                           name="my_catalog_sku"
                           value="<?php echo esc_attr( $sku ); ?>" />
                </td>
            </tr>
            <tr>
                <th scope="row">
                    <label for="my-catalog-stock-status"><?php esc_html_e( 'Availability', 'my-catalog' ); ?></label>
                </th>
                <td>
                    <select id="my-catalog-stock-status" name="my_catalog_stock_status">
                        <?php foreach ( self::get_stock_statuses() as $value => $label ) : ?>
                            <option value="<?php echo esc_attr( $value ); ?>"
                                <?php selected( $stock_status, $value ); ?>>
                                <?php echo esc_html( $label ); ?>
                            </option>
                        <?php endforeach; ?>
                    </select>
                </td>
            </tr>
            <tr>
                <th scope="row">
                    <label for="my-catalog-weight"><?php esc_html_e( 'Weight', 'my-catalog' ); ?></label>
                </th>
                <td>
                    <input type="text" class="regular-text" id="my-catalog-weight"
                           name="my_catalog_weight"
                           value="<?php echo esc_attr( $weight ); ?>" />
                </td>
            </tr>
            <tr>
                <th scope="row">
                    <label for="my-catalog-external-url"><?php esc_html_e( 'External Store URL', 'my-catalog' ); ?></label>
                </th>
                <td>
                    <input type="url" class="regular-text" id="my-catalog-external-url"
                           name="my_catalog_external_url"
                           value="<?php echo esc_attr( $external_url ); ?>" />
                </td>
            </tr>
        </tbody>
    </table>
    <?php
}
```

Статус наличия выбирается из фиксированного набора значений, определённых в статическом методе `get_stock_statuses()`:

```php
public static function get_stock_statuses() {
    return array(
        'in_stock'     => __( 'In stock', 'my-catalog' ),
        'out_of_stock' => __( 'Out of stock', 'my-catalog' ),
        'preorder'     => __( 'Pre-order', 'my-catalog' ),
    );
}
```

Категории и атрибуты товаров создаются заранее через таксономии «Product Categories» (иерархическая) и «Product Attributes» (неиерархическая), зарегистрированные в `register_content_types()`. При сохранении товара данные проходят валидацию: проверяются nonce, защита от автосохранений, значения санитаризуются через `sanitize_text_field()`, `sanitize_key()`, `esc_url_raw()` и кастомный `sanitize_decimal_field()`.

### 3.10.2 Размещение таблицы на странице (Начальный вид)

Автор добавляет на страницу шорткод `[product_table limit="5" columns="image,title,price,category,stock"]`. На фронтенде отображается пустая таблица с надписью «No products found» (Рисунок 3.6) — данные загружаются асинхронно через REST API после инициализации DataTables на стороне клиента.

[Место для Рисунка 3.6 — Скриншот пустой таблицы с сообщением «No products found»]

Шорткод обрабатывается методом `render_shortcode()` класса `My_Catalog_Product_Table`. Атрибуты парсятся через `shortcode_atts()`, колонки фильтруются через пересечение с доступными, формируется JSON-конфигурация:

```php
$config = array(
    'restUrl'      => rest_url( 'my-catalog/v1/product-table' ),
    'baseCategory' => sanitize_text_field( $atts['category'] ),
    'baseTag'      => sanitize_text_field( $atts['tag'] ),
    'columns'      => $columns,
    'pageLength'   => max( 1, absint( $atts['limit'] ) ),
    'search'       => filter_var( $atts['search'], FILTER_VALIDATE_BOOLEAN ),
    'defaultOrder' => $this->get_default_order_for_columns( $columns ),
);
```

Конфигурация записывается в data-атрибут контейнера. Таблица рендерится с пустым телом — единственная строка содержит сообщение об отсутствии товаров на всю ширину колонок:

```php
<div class="my-catalog-product-table" id="<?php echo esc_attr( $instance_id ); ?>"
     data-config="<?php echo esc_attr( wp_json_encode( $config ) ); ?>">
    <div class="my-catalog-product-table__frame">
        <table class="display responsive nowrap" style="width:100%">
            <thead>
                <tr>
                    <?php foreach ( $columns as $column ) : ?>
                        <th><?php echo esc_html( $column['label'] ); ?></th>
                    <?php endforeach; ?>
                </tr>
            </thead>
            <tbody>
                <tr>
                    <td colspan="<?php echo esc_attr( count( $columns ) ); ?>">
                        <?php echo esc_html( $atts['empty_message'] ); ?>
                    </td>
                </tr>
            </tbody>
        </table>
    </div>
</div>
```

После загрузки страницы JavaScript-код читает `data-config`, инициализирует DataTables в серверном режиме и отправляет первый AJAX-запрос к REST endpoint `GET /my-catalog/v1/product-table`. Если товаров в базе нет, API возвращает пустой массив `data`, и таблица отображает сообщение «No products found», переданное DataTables.

### 3.10.3 Работа с таблицей

После загрузки данных таблица становится интерактивной (Рисунок 3.7). Посетитель сайта может выполнять следующие действия:

- **Поиск** — строка поиска фильтрует записи по заголовку, содержанию, мета-полям (цена, артикул, статус, вес) и названиям терминов таксономий. Поиск реализован через кастомный SQL в `add_search_clauses()`, который модифицирует `posts_clauses` и выполняет `LIKE`-запрос с `OR` по всем полям.
- **Сортировка** — по заголовку, цене, артикулу, статусу наличия и весу. Порядок сортировки задаётся через параметры `order_column` и `order_dir`, передаваемые в REST endpoint. Каждая колонка имеет маппинг на соответствующий `orderby` (например, `price → meta_value_num`).
- **Пагинация** — разбивка на страницы с заданным в шорткоде лимитом (`limit`), навигация по страницам без перезагрузки.

[Место для Рисунка 3.7 — Скриншот таблицы с загруженными данными, поиском и пагинацией]

DataTables обрабатывает все взаимодействия на стороне клиента, отправляя AJAX-запросы к `GET /my-catalog/v1/product-table`. Параметры запроса формируются в JavaScript-функции `data` внутри `ajax`-конфигурации:

```javascript
ajax: {
    url: config.restUrl || myCatalogProductTable.restUrl,
    type: 'GET',
    data: function (data) {
        var order = data.order && data.order[0]
            ? data.order[0]
            : { column: 0, dir: 'asc' };
        var orderColumn = columns[order.column]
            ? columns[order.column].name
            : 'title';
        return {
            draw: data.draw,
            start: data.start,
            length: data.length,
            search: data.search ? data.search.value : '',
            order_column: orderColumn,
            order_dir: order.dir || 'asc',
            category: getFilterValue($category),
            tag: '',
            price_min: getPriceFilterValue($priceMin, 'min'),
            price_max: getPriceFilterValue($priceMax, 'max'),
            base_category: config.baseCategory || '',
            base_tag: config.baseTag || '',
            columns: columns.map(function (col) { return col.name; }),
        };
    },
},
```

Если на странице присутствует блок фильтров (шорткод `[product_filters]` или Gutenberg-блок «Product Filters»), JavaScript находит связанную с ним таблицу через атрибут `data-target` или применяет фильтры ко всем таблицам, если `target` не указан. При изменении категории в выпадающем списке или перемещении бегунков ценового диапазона вызывается `table.ajax.reload()`, отправляющий новый запрос с обновлёнными параметрами `category`, `price_min`, `price_max`:

```javascript
$category.on('change', function () { table.ajax.reload(); });

$priceRanges.on('input change', 'input', function () {
    // ... синхронизация полей ...
    table.ajax.reload();
});
```

На серверной стороне REST endpoint обрабатывает запрос в `handle_rest_request()`. Метод строит два `WP_Query`: первый — для подсчёта общего количества записей без фильтров (`recordsTotal`), второй — с применением всех переданных фильтров, поиска, сортировки и пагинации (`recordsFiltered`, `data`). Ответ возвращается в формате DataTables protocol:

```php
return rest_ensure_response(
    array(
        'draw'            => $draw,
        'recordsTotal'    => absint( $total_query->found_posts ),
        'recordsFiltered' => absint( $filtered_query->found_posts ),
        'data'            => $data,
    )
);
```

Каждая строка товара форматируется в `format_product_row()`: для колонки `image` выводится миниатюра (или плейсхолдер), для `title` — ссылка на страницу товара, для `price` — число с двумя знаками после запятой, для `stock` — локализованный статус наличия, для `external` — кнопка «Buy» с ссылкой на внешний магазин.

### 3.10.4 Карусель новостей (Итоговый вид)

Автор добавляет в запись Gutenberg-блок «News Carousel» через редактор блоков (Рисунок 3.8). В панели инспектора настраиваются параметры: лимит — 6 новостей, категория — «Все» (пустое значение), автовоспроизведение — включено, задержка — 5000 мс.

[Место для Рисунка 3.8 — Скриншот редактора с блоком «News Carousel» и открытой панелью инспектора]

Атрибуты блока описаны в `block.json`:

```json
{
  "name": "my-catalog/news-carousel",
  "attributes": {
    "limit":         { "type": "number", "default": 6 },
    "category":      { "type": "string", "default": "" },
    "slidesPerView": { "type": "number", "default": 1 },
    "autoplay":      { "type": "boolean", "default": true },
    "autoplayDelay": { "type": "number", "default": 5000 }
  }
}
```

На фронтенде PHP-метод `render()` класса `My_Catalog_News_Carousel` выполняет `WP_Query` для получения опубликованных записей типа `news`. Категория фильтруется через `tax_query` по таксономии `news_category`:

```php
$query_args = array(
    'post_type'      => My_Catalog_Core::NEWS_POST_TYPE,
    'post_status'    => 'publish',
    'posts_per_page' => max( 1, absint( $atts['limit'] ) ),
);

if ( ! empty( $atts['category'] ) ) {
    $field = is_numeric( $atts['category'] ) ? 'term_id' : 'slug';
    $query_args['tax_query'] = array(
        array(
            'taxonomy' => My_Catalog_Core::NEWS_CATEGORY_TAXONOMY,
            'field'    => $field,
            'terms'    => is_numeric( $atts['category'] )
                ? absint( $atts['category'] )
                : sanitize_title( $atts['category'] ),
        ),
    );
}

$query = new WP_Query( $query_args );
```

Далее генерируется Swiper-карусель с навигационными стрелками и пагинацией. Конфигурация передаётся через data-атрибут:

```php
$config = array(
    'slidesPerView' => 1,
    'autoplay'      => filter_var( $atts['autoplay'], FILTER_VALIDATE_BOOLEAN ),
    'autoplayDelay' => max( 1000, absint( $atts['autoplay_delay'] ) ),
    'slideCount'    => (int) $query->post_count,
);
```

HTML-разметка карусели включает три основные зоны: контейнер слайдов (swiper-wrapper), навигационные стрелки и пагинацию:

```php
<div class="my-catalog-news-carousel" id="<?php echo esc_attr( $instance_id ); ?>"
     data-config="<?php echo esc_attr( wp_json_encode( $config ) ); ?>">
    <div class="swiper">
        <div class="swiper-wrapper">
            <?php foreach ( $query->posts as $post ) {
                echo $this->render_slide( $post );
            } ?>
        </div>
    </div>
    <div class="my-catalog-news-carousel__navigation">
        <button type="button" class="my-catalog-news-carousel__arrow--prev"
                aria-label="<?php esc_attr_e( 'Previous slide', 'my-catalog' ); ?>">
            <span aria-hidden="true">&larr;</span>
        </button>
        <div class="swiper-pagination"></div>
        <button type="button" class="my-catalog-news-carousel__arrow--next"
                aria-label="<?php esc_attr_e( 'Next slide', 'my-catalog' ); ?>">
            <span aria-hidden="true">&rarr;</span>
        </button>
    </div>
</div>
```

Каждый слайд рендерится через метод `render_slide()`, который загружает шаблон `templates/news-slide.php`. Шаблон выводит изображение, заголовок новости, отрывок текста и кнопку «Read More»:

```php
// render_slide() — подготовка данных
$data = array(
    'title'       => get_the_title( $post ),
    'excerpt'     => has_excerpt( $post )
        ? $post->post_excerpt
        : wp_trim_words( wp_strip_all_tags( $post->post_content ), 24 ),
    'image_html'  => get_the_post_thumbnail( $post, 'large', array( 'class' => 'my-catalog-news-carousel__image' ) ),
    'link'        => $link, // кастомный URL из мета-поля или permalink
    'button_text' => __( 'Read More', 'my-catalog' ),
);
include $template;
```

```php
<!-- templates/news-slide.php -->
<article class="swiper-slide my-catalog-news-carousel__slide">
    <div class="my-catalog-news-carousel__media">
        <?php if ( ! empty( $data['image_html'] ) ) : ?>
            <?php echo $data['image_html']; ?>
        <?php endif; ?>
    </div>
    <div class="my-catalog-news-carousel__body">
        <h3 class="my-catalog-news-carousel__title"><?php echo esc_html( $data['title'] ); ?></h3>
        <p class="my-catalog-news-carousel__excerpt"><?php echo esc_html( $data['excerpt'] ); ?></p>
        <a class="my-catalog-news-carousel__link" href="<?php echo esc_url( $data['link'] ); ?>">
            <?php echo esc_html( $data['button_text'] ); ?>
        </a>
    </div>
</article>
```

[Место для Рисунка 3.9 — Скриншот готовой Swiper-карусели на фронтенде с навигацией]

JavaScript инициализирует Swiper с поддержкой цикличного переключения, автовоспроизведения (если включено и слайдов больше одного), навигации стрелками и кликабельной пагинации:

```javascript
new Swiper(carousel.querySelector('.swiper'), {
    slidesPerView: 1,
    spaceBetween: 24,
    loop: slideCount > 1,
    grabCursor: true,
    autoplay: autoplayEnabled
        ? {
              delay: Math.max(1000, parseInt(config.autoplayDelay, 10) || 5000),
              disableOnInteraction: false,
              pauseOnMouseEnter: true,
          }
        : false,
    navigation: {
        nextEl: carousel.querySelector('.my-catalog-news-carousel__arrow--next'),
        prevEl: carousel.querySelector('.my-catalog-news-carousel__arrow--prev'),
    },
    pagination: {
        el: carousel.querySelector('.swiper-pagination'),
        clickable: true,
    },
});
```

Тема может переопределить шаблон слайда, разместив файл `my-catalog/news-slide.php` или `my-catalog-news-slide.php` в корне темы. Плагин проверяет их наличие через `locate_template()` в порядке приоритета; если файл не найден, используется встроенный шаблон из директории `templates/`.

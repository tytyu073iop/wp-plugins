## 3.1. Экранные формы и пользовательский интерфейс

Первое, что видит автор после активации плагина на своём сайте, — это страница настроек в панели управления. На фронтенде же отображается результат работы шорткодов и Gutenberg-блоков, которые автор размещает в записях и на страницах. Рассмотрим каждую из экранных форм подробнее.

### 3.1.1 Административная страница настроек

После активации плагина в меню «Settings» появляется пункт «My Catalog» (Рисунок 3.1). Страница настроек реализована через стандартный Settings API WordPress и позволяет автору выбрать колонки таблицы товаров, отображаемые по умолчанию.

[Место для Рисунка 3.1 – Скриншот страницы настроек «My Catalog»]

Регистрация страницы выполняется в методе `register_settings_page()` класса `My_Catalog_Product_Table`. В нём вызывается функция `add_options_page()`, которая добавляет пункт в меню «Settings» и связывает его с методом `render_settings_page()`, отвечающим за вывод HTML-разметки:

```php
public function register_settings_page() {
    add_options_page(
        __( 'My Catalog Settings', 'my-catalog' ),
        __( 'My Catalog', 'my-catalog' ),
        'manage_options',
        'my-catalog-settings',
        array( $this, 'render_settings_page' )
    );
}
```

Набор доступных колонок централизованно описан в методе `get_available_columns()`. Каждая колонка имеет ключ, отображаемое имя и флаг возможности сортировки:

```php
private function get_available_columns() {
    return array(
        'image'      => array(
            'label'     => __( 'Image', 'my-catalog' ),
            'orderable' => false,
        ),
        'title'      => array(
            'label'     => __( 'Name', 'my-catalog' ),
            'orderable' => true,
        ),
        'price'      => array(
            'label'     => __( 'Price', 'my-catalog' ),
            'orderable' => true,
        ),
        'category'   => array(
            'label'     => __( 'Category', 'my-catalog' ),
            'orderable' => false,
        ),
        'attributes' => array(
            'label'     => __( 'Attributes', 'my-catalog' ),
            'orderable' => false,
        ),
        'sku'        => array(
            'label'     => __( 'SKU', 'my-catalog' ),
            'orderable' => true,
        ),
        'stock'      => array(
            'label'     => __( 'Availability', 'my-catalog' ),
            'orderable' => true,
        ),
        'weight'     => array(
            'label'     => __( 'Weight', 'my-catalog' ),
            'orderable' => true,
        ),
        'external'   => array(
            'label'     => __( 'Buy', 'my-catalog' ),
            'orderable' => false,
        ),
    );
}
```

HTML-разметка страницы генерируется в `render_settings_page()` (Приложение А). Метод перебирает массив доступных колонок и для каждой выводит элемент `<input type="checkbox">`. Если колонка уже выбрана (значение есть в сохранённой опции), атрибут `checked` проставляется автоматически. Внизу формы расположена стандартная кнопка «Save Changes».

Регистрация опции для хранения выбранных колонок выполняется через `register_setting()`. Тип данных — `array`, значение по умолчанию — пять колонок (`image`, `title`, `price`, `category`, `sku`). Для санитаризации используется callback `sanitize_columns()`, который вычисляет пересечение переданных ключей с ключами `get_available_columns()`, отбрасывая все невалидные значения:

```php
public function register_settings() {
    register_setting(
        'my_catalog_settings',
        self::OPTION_COLUMNS,
        array(
            'type'              => 'array',
            'sanitize_callback' => array( $this, 'sanitize_columns' ),
            'default'           => $this->get_default_columns(),
        )
    );
}

public function sanitize_columns( $columns ) {
    $available = array_keys( $this->get_available_columns() );
    $columns   = is_array( $columns )
        ? array_map( 'sanitize_key', $columns )
        : array();
    $columns   = array_values( array_intersect( $columns, $available ) );

    return ! empty( $columns ) ? $columns : $this->get_default_columns();
}
```

### 3.1.2 Фронтенд: шорткод таблицы товаров

На фронтенде автор размещает шорткод `[product_table]` в теле произвольной записи или на странице (Рисунок 3.2). Шорткод поддерживает следующие атрибуты:

- `limit` — количество записей на странице (по умолчанию 10);
- `category` — фильтрация по категории товара (slug или ID);
- `tag` — фильтрация по атрибуту/тегу;
- `columns` — список отображаемых колонок через запятую (переопределяет настройки);
- `search` — включение или отключение строки поиска;
- `table_id` — кастомный идентификатор для связи с фильтрами;
- `empty_message` — сообщение при отсутствии товаров.

[Место для Рисунка 3.2 – Скриншот таблицы товаров на фронтенде]

Рендеринг шорткода выполняется методом `render_shortcode()` класса `My_Catalog_Product_Table`. Вначале атрибуты сливаются со значениями по умолчанию через `shortcode_atts()`. Затем определяется набор колонок: если атрибут `columns` не указан, используются сохранённые в настройках; в противном случае переданный список парсится, санитаризуется и пересекается с доступными колонками:

```php
public function render_shortcode( $atts ) {
    $atts = shortcode_atts(
        array(
            'limit'         => 10,
            'category'      => '',
            'tag'           => '',
            'columns'       => '',
            'search'        => 'true',
            'empty_message' => __( 'No products found.', 'my-catalog' ),
            'table_id'      => '',
        ),
        $atts,
        'product_table'
    );

    $column_keys = $this->parse_columns_attribute( $atts['columns'] );
    $columns     = $this->prepare_columns_for_frontend( $column_keys );
    $instance_id = sanitize_html_class( $atts['table_id'] )
        ?: 'my-catalog-product-table-' . ++self::$instance;

    wp_enqueue_style( 'my-catalog-product-table' );
    wp_enqueue_script( 'my-catalog-product-table' );
```

Далее формируется JSON-конфигурация, которая записывается в data-атрибут контейнера. Эта конфигурация содержит URL REST-эндпоинта, базовые фильтры, список колонок, длину страницы и порядок сортировки по умолчанию:

```php
    $config = array(
        'restUrl'       => rest_url( 'my-catalog/v1/product-table' ),
        'baseCategory'  => sanitize_text_field( $atts['category'] ),
        'baseTag'       => sanitize_text_field( $atts['tag'] ),
        'columns'       => $columns,
        'pageLength'    => max( 1, absint( $atts['limit'] ) ),
        'search'        => filter_var( $atts['search'], FILTER_VALIDATE_BOOLEAN ),
        'defaultOrder'  => $this->get_default_order_for_columns( $columns ),
    );
```

Генерация HTML-разметки использует буферизацию вывода (`ob_start()` / `ob_get_clean()`). Таблица рендерится с пустым телом — реальные данные загружаются асинхронно после инициализации DataTables на стороне клиента:

```php
    ob_start();
    ?>
    <div class="my-catalog-product-table" id="<?php echo esc_attr( $instance_id ); ?>"
         data-config="<?php echo esc_attr( wp_json_encode( $config ) ); ?>">
        <div class="my-catalog-product-table__frame">
            <table class="display responsive nowrap" style="width:100%">
                <thead>
                    <tr>
                        <?php foreach ( $columns as $column ) : ?>
                            <th><?php echo esc_html( $column['label'] ); ?></th>
                        <?php endforeach; ?>
                    </tr>
                </thead>
                <tbody>
                    <tr>
                        <td colspan="<?php echo esc_attr( count( $columns ) ); ?>">
                            <?php echo esc_html( $atts['empty_message'] ); ?>
                        </td>
                    </tr>
                </tbody>
            </table>
        </div>
    </div>
    <?php

    return (string) ob_get_clean();
}
```

После загрузки страницы JavaScript-код читает `data-config`, инициализирует DataTables с серверным режимом обработки и отправляет AJAX-запрос к endpoint `GET /my-catalog/v1/product-table`. Ответ содержит данные в формате DataTables protocol (`draw`, `recordsTotal`, `recordsFiltered`, `data`). Таблица автоматически отображает полученные строки с поддержкой сортировки, поиска и пагинации без перезагрузки страницы.

Для подключения DataTables используются зарегистрированные стили и скрипты, загружаемые с CDN:

```php

### 3.1.3 Фронтенд: Gutenberg-блок «Карусель новостей»

В редакторе блоков Gutenberg автору доступен блок «News Carousel» (Рисунок 3.3). Блок использует API v3 и является динамическим — метод `save()` возвращает `null`, а разметка генерируется на сервере через `render_callback`.

[Место для Рисунка 3.3 – Скриншот блока «News Carousel» в редакторе]

Регистрация блока выполняется в координаторе плагина `My_Catalog_Plugin::register_block()`. Если файл `block.json` существует, блок регистрируется с callback'ом на `My_Catalog_News_Carousel::render_block()`:

```php
public function register_block() {
    if ( file_exists( MY_CATALOG_PATH . '/block.json' ) ) {
        register_block_type(
            MY_CATALOG_PATH,
            array(
                'render_callback' => array( $this->news_carousel, 'render_block' ),
            )
        );
    }
}
```

Атрибуты блока описаны в `block.json`:

```json
{
  "$schema": "https://schemas.wp.org/trunk/block.json",
  "apiVersion": 3,
  "name": "my-catalog/news-carousel",
  "title": "News Carousel",
  "category": "widgets",
  "icon": "slides",
  "attributes": {
    "limit":         { "type": "number", "default": 6 },
    "category":      { "type": "string", "default": "" },
    "slidesPerView": { "type": "number", "default": 1 },
    "autoplay":      { "type": "boolean", "default": true },
    "autoplayDelay": { "type": "number", "default": 5000 }
  }
}
```

В редакторе блок отображается через компонент `ServerSideRender` (файл `src/index.js`). Панель инспектора (`InspectorControls`) содержит настройки: количество новостей (`RangeControl`), категория (`SelectControl` с динамической загрузкой таксономии через `useSelect`), автовоспроизведение (`ToggleControl`) и задержка (`RangeControl`, отображается только при включённом автовоспроизведении):

```jsx
function NewsCarouselEdit( { attributes, setAttributes } ) {
    const { limit, category, autoplay, autoplayDelay } = attributes;
    const categories = useSelect( ( select ) =>
        select( 'core' ).getEntityRecords( 'taxonomy', 'news_category', {
            per_page: -1, hide_empty: false,
        } ), []
    );
    const categoryOptions = [
        { label: __( 'All categories', 'my-catalog' ), value: '' },
        ...( categories || [] ).map( ( term ) => ( {
            label: term.name, value: term.slug,
        } ) ),
    ];

    return (
        <>
            <InspectorControls>
                <PanelBody title={ __( 'Carousel settings', 'my-catalog' ) }>
                    <RangeControl label={ __( 'News items', 'my-catalog' ) }
                        value={ limit }
                        onChange={ ( v ) => setAttributes( { limit: v || 1 } ) }
                        min={ 1 } max={ 12 } />
                    { categories === null ? (
                        <Spinner />
                    ) : (
                        <SelectControl label={ __( 'Category', 'my-catalog' ) }
                            value={ category } options={ categoryOptions }
                            onChange={ ( v ) => setAttributes( { category: v } ) } />
                    ) }
                    <ToggleControl label={ __( 'Autoplay', 'my-catalog' ) }
                        checked={ autoplay }
                        onChange={ ( v ) => setAttributes( { autoplay: v } ) } />
                    { autoplay && (
                        <RangeControl label={ __( 'Autoplay delay (ms)', 'my-catalog' ) }
                            value={ autoplayDelay }
                            onChange={ ( v ) => setAttributes( { autoplayDelay: v || 1000 } ) }
                            min={ 1000 } max={ 10000 } step={ 500 } />
                    ) }
                </PanelBody>
            </InspectorControls>
            <div { ...blockProps }>
                <ServerSideRender block={ metadata.name } attributes={ attributes } />
            </div>
        </>
    );
}
```

На фронтенде PHP-метод `render()` класса `My_Catalog_News_Carousel` принимает атрибуты блока, маппит camelCase-ключи в snake_case (например `autoplayDelay` → `autoplay_delay`) и выполняет `WP_Query` для получения опубликованных новостей. Если новостей нет, выводится сообщение «No news items found.»:

```php
public function render( $atts ) {
    $query_args = array(
        'post_type'      => My_Catalog_Core::NEWS_POST_TYPE,
        'post_status'    => 'publish',
        'posts_per_page' => max( 1, absint( $atts['limit'] ) ),
        'ignore_sticky_posts' => true,
    );

    if ( ! empty( $atts['category'] ) ) {
        $field = is_numeric( $atts['category'] ) ? 'term_id' : 'slug';
        $query_args['tax_query'] = array(
            array(
                'taxonomy' => My_Catalog_Core::NEWS_CATEGORY_TAXONOMY,
                'field'    => $field,
                'terms'    => is_numeric( $atts['category'] )
                    ? absint( $atts['category'] )
                    : sanitize_title( $atts['category'] ),
            ),
        );
    }

    $query = new WP_Query( $query_args );

    if ( ! $query->have_posts() ) {
        return '<div class="my-catalog-news-carousel__empty">'
            . esc_html__( 'No news items found.', 'my-catalog' )
            . '</div>';
    }
```

Далее подключаются стили и скрипты Swiper, генерируется контейнер карусели с data-конфигурацией и наполняется слайдами:

```php

                } ?>
            </div>
        </div>
        <div class="my-catalog-news-carousel__navigation">
            <button type="button" class="my-catalog-news-carousel__arrow--prev"
                    aria-label="<?php esc_attr_e( 'Previous slide', 'my-catalog' ); ?>">

Шаблон выводит изображение (через `the_post_thumbnail()`), заголовок, отрывок текста и кнопку «Read More» (Приложение Б).

### 3.1.4 Фронтенд: блок фильтров товаров

Блок «Product Filters» (Рисунок 3.4) позволяет автору разместить фильтры по категории и диапазону цены отдельно от таблицы товаров. Атрибуты блока описаны в `blocks/product-filters/block.json`:

[Место для Рисунка 3.4 – Скриншот блока «Product Filters» на фронтенде]

```json
{
  "name": "my-catalog/product-filters",
  "apiVersion": 3,
  "title": "Product Filters",
  "category": "widgets",
  "icon": "filter",
  "attributes": {
    "target":       { "type": "string", "default": "" },
    "showCategory": { "type": "boolean", "default": true },
    "showPrice":    { "type": "boolean", "default": true }
  }
}
```

Фильтры связываются с таблицей через атрибут `target` — ID контейнера таблицы (значение атрибута `table_id` шорткода `[product_table]`). Если `target` не указан, JavaScript применяет фильтры ко всем таблицам на странице.

На стороне сервера рендеринг фильтров выполняется методом `render_filter_controls()` класса `My_Catalog_Product_Table`. Для категорий генерируется выпадающий список (`<select>`) с опциями, загружаемыми из таксономии `product_cat`. Для цен — двойной ползунок с синхронизированными числовыми полями. Границы ценового диапазона вычисляются на основе существующих товаров через SQL-запрос:

```php
private function get_price_filter_bounds() {
    global $wpdb;

    $bounds = $wpdb->get_row(
        $wpdb->prepare(
            "SELECT MIN(CAST(meta_value AS DECIMAL(20,4))) AS min_price,
                    MAX(CAST(meta_value AS DECIMAL(20,4))) AS max_price
             FROM {$wpdb->postmeta}
             WHERE meta_key = %s
               AND meta_value != ''
               AND meta_value REGEXP '^[0-9]+(\\\\.[0-9]+)?$'",
            My_Catalog_Core::PRODUCT_META_PRICE
        ),
        ARRAY_A
    );

    $min = isset( $bounds['min_price'] ) ? floor( (float) $bounds['min_price'] ) : 0;
    $max = isset( $bounds['max_price'] ) ? ceil( (float) $bounds['max_price'] ) : 0;

    if ( $max <= $min ) {
        $max = $min + 1;
    }

    return array( 'min' => $min, 'max' => $max );
}
```

При изменении любого фильтра JavaScript-обработчик отправляет новый запрос к REST API `GET /my-catalog/v1/product-table` с обновлёнными параметрами `category`, `price_min` и `price_max`. Таблица перерисовывается без перезагрузки страницы.

## 3.7. Сборка frontend-ресурсов

### 3.7.1 Webpack через `@wordpress/scripts`

Исходные файлы Gutenberg-блоков находятся в директории `src/` и включают три файла:

```
src/
  index.js      -- точка входа: регистрация блоков, React-компоненты Edit
  style.scss    -- стили фронтенда для карусели новостей
  editor.scss   -- стили для редактора Gutenberg
```

Сборка выполняется через `@wordpress/scripts` (версия 30.24.0), который предоставляет стандартную конфигурацию webpack без необходимости создавать собственный `webpack.config.js`. Кастомный флаг `--webpack-copy-php` включает копирование PHP-файлов, связанных с `block.json`, в директорию сборки:

```json
{
  "scripts": {
    "start": "wp-scripts start --webpack-copy-php",
    "build": "wp-scripts build --webpack-copy-php"
  },
  "devDependencies": {
    "@wordpress/env": "^10.19.0",
    "@wordpress/scripts": "^30.24.0"
  }
}
```

Команда `npm run build` собирает исходные файлы и помещает результат в директорию `build/`, которая указана в `.gitignore` и не попадает в репозиторий. После клонирования или переключения ветки необходимо выполнить сборку:

```sh
npm install
npm run build
```

`npm run start` запускает webpack в режиме наблюдения (`watch mode`), автоматически пересобирая файлы при изменениях в процессе разработки.

### 3.7.2 Подключение библиотек через CDN

Сторонние библиотеки не включаются в webpack-бандл, а подключаются через CDN стандартными средствами WordPress. Для каждой библиотеки регистрируются CSS и JavaScript через `wp_register_style()` / `wp_register_script()` с указанием версии и зависимостей.

**DataTables 1.13.8** (с дополнением Responsive 2.5.0) регистрируется в методе `register_assets()` класса `My_Catalog_Product_Table`:

```php
public function register_assets() {
    wp_register_style(
        'my-catalog-datatables',
        'https://cdn.datatables.net/1.13.8/css/jquery.dataTables.min.css',
        array(),
        '1.13.8'
    );

    wp_register_style(
        'my-catalog-datatables-responsive',
        'https://cdn.datatables.net/responsive/2.5.0/css/responsive.dataTables.min.css',
        array( 'my-catalog-datatables' ),
        '2.5.0'
    );

    wp_register_style(
        'my-catalog-product-table',
        MY_CATALOG_URL . 'assets/css/product-table.css',
        array( 'my-catalog-datatables-responsive' ),
        MY_CATALOG_VERSION
    );

    wp_register_script(
        'my-catalog-datatables',
        'https://cdn.datatables.net/1.13.8/js/jquery.dataTables.min.js',
        array( 'jquery' ),
        '1.13.8',
        true
    );

    wp_register_script(
        'my-catalog-datatables-responsive',
        'https://cdn.datatables.net/responsive/2.5.0/js/dataTables.responsive.min.js',
        array( 'my-catalog-datatables' ),
        '2.5.0',
        true
    );

    wp_register_script(
        'my-catalog-product-table',
        MY_CATALOG_URL . 'assets/js/product-table.js',
        array( 'jquery', 'my-catalog-datatables-responsive' ),
        MY_CATALOG_VERSION,
        true
    );

    wp_localize_script(
        'my-catalog-product-table',
        'myCatalogProductTable',
        array(
            'restUrl' => esc_url_raw( rest_url( 'my-catalog/v1/product-table' ) ),
        )
    );
}
```

Цепочка зависимостей:

```
CSS: my-catalog-datatables
       └─ my-catalog-datatables-responsive
              └─ my-catalog-product-table (локальный)

JS:  jquery
       └─ my-catalog-datatables
              └─ my-catalog-datatables-responsive
                     └─ my-catalog-product-table (локальный)
```

**Swiper 11.1.4** регистрируется в методе `register_assets()` класса `My_Catalog_News_Carousel`. Swiper не имеет зависимостей (в отличие от DataTables, который требует jQuery):

```php
public function register_assets() {
    wp_register_style(
        'my-catalog-swiper',
        'https://cdn.jsdelivr.net/npm/swiper@11/swiper-bundle.min.css',
        array(),
        '11.1.4'
    );

    wp_register_style(
        'my-catalog-news-carousel',
        MY_CATALOG_URL . 'assets/css/news-carousel.css',
        array( 'my-catalog-swiper' ),
        MY_CATALOG_VERSION
    );

    wp_register_script(
        'my-catalog-swiper',
        'https://cdn.jsdelivr.net/npm/swiper@11/swiper-bundle.min.js',
        array(),
        '11.1.4',
        true
    );

    wp_register_script(
        'my-catalog-news-carousel',
        MY_CATALOG_URL . 'assets/js/news-carousel.js',
        array( 'my-catalog-swiper' ),
        MY_CATALOG_VERSION,
        true
    );
}
```

Цепочка зависимостей:

```
CSS: my-catalog-swiper (CDN)
       └─ my-catalog-news-carousel (локальный)

JS:  my-catalog-swiper (CDN)
       └─ my-catalog-news-carousel (локальный)
```

Оба метода `register_assets()` вызываются на хуке `wp_enqueue_scripts`, что гарантирует регистрацию ресурсов до момента их запроса на фронтенде. Фактическое подключение (`wp_enqueue_style()` / `wp_enqueue_script()`) происходит непосредственно в методах рендеринга — `render_shortcode()` для таблицы и `render()` для карусели новостей. Это означает, что CSS и JS загружаются только на тех страницах, где присутствует соответствующий шорткод или блок.

Таблица зарегистрированных CDN-ресурсов:

| Библиотека | Версия | CSS | JS | Зависимости JS |
|---|---|---|---|---|
| DataTables | 1.13.8 | `cdn.datatables.net/1.13.8/css/...` | `cdn.datatables.net/1.13.8/js/...` | jQuery |
| DataTables Responsive | 2.5.0 | `cdn.datatables.net/responsive/2.5.0/css/...` | `cdn.datatables.net/responsive/2.5.0/js/...` | DataTables |
| Swiper | 11.1.4 | `cdn.jsdelivr.net/npm/swiper@11/swiper-bundle.min.css` | `cdn.jsdelivr.net/npm/swiper@11/swiper-bundle.min.js` | нет |

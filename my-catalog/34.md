## 3.4. Хранение данных

### 3.4.1 Post Meta

Данные товаров и новостей хранятся в стандартной таблице `wp_postmeta` через API `update_post_meta()` / `get_post_meta()`. Все мета-ключи используют префикс `_my_catalog_` (символ подчёркивания в начале делает их защищёнными — скрытыми из стандартного UI произвольных полей WordPress).

Мета-ключи централизованно описаны в виде констант класса `My_Catalog_Core`:

```php
// Мета-поля товаров
const PRODUCT_META_PRICE       = '_my_catalog_price';
const PRODUCT_META_SKU         = '_my_catalog_sku';
const PRODUCT_META_STOCK       = '_my_catalog_stock_status';
const PRODUCT_META_WEIGHT      = '_my_catalog_weight';
const PRODUCT_META_EXTERNAL_URL = '_my_catalog_external_url';

// Мета-поля новостей
const NEWS_META_READ_MORE_URL  = '_my_catalog_read_more_url';
```

Типы хранимых значений:
- `string` — цена, артикул, вес (десятичные числа, сохраняются как строки);
- `string` — статус наличия (фиксированный набор: `in_stock`, `out_of_stock`, `preorder`);
- `string` — URL внешней ссылки;
- `string` — URL кнопки «Read More» для новостей.

Сохранение выполняется через хуки `save_post_product` и `save_post_news`, зарегистрированные в конструкторе:

```php
add_action( 'save_post_' . self::PRODUCT_POST_TYPE, array( $this, 'save_product_meta' ) );
add_action( 'save_post_' . self::NEWS_POST_TYPE,    array( $this, 'save_news_meta' ) );
```

Метод `save_product_meta()` проверяет nonce, защиту от автосохранений и ревизий, затем санитаризует каждое поле и сохраняет через `update_post_meta()`:

```php
public function save_product_meta( $post_id ) {
    if ( ! $this->can_save_post( $post_id, 'my_catalog_product_meta_nonce', 'my_catalog_save_product_meta' ) ) {
        return;
    }

    $stock_status = isset( $_POST['my_catalog_stock_status'] )
        ? sanitize_key( wp_unslash( $_POST['my_catalog_stock_status'] ) )
        : 'in_stock';
    $stock_map    = self::get_stock_statuses();

    update_post_meta( $post_id, self::PRODUCT_META_PRICE,
        $this->sanitize_decimal_field( 'my_catalog_price' ) );
    update_post_meta( $post_id, self::PRODUCT_META_SKU,
        isset( $_POST['my_catalog_sku'] )
            ? sanitize_text_field( wp_unslash( $_POST['my_catalog_sku'] ) )
            : '' );
    update_post_meta( $post_id, self::PRODUCT_META_STOCK,
        isset( $stock_map[ $stock_status ] ) ? $stock_status : 'in_stock' );
    update_post_meta( $post_id, self::PRODUCT_META_WEIGHT,
        $this->sanitize_decimal_field( 'my_catalog_weight' ) );
    update_post_meta( $post_id, self::PRODUCT_META_EXTERNAL_URL,
        isset( $_POST['my_catalog_external_url'] )
            ? esc_url_raw( wp_unslash( $_POST['my_catalog_external_url'] ) )
            : '' );
}
```

Для десятичных полей (цена, вес) используется метод `sanitize_decimal_field()`, который заменяет запятую на точку и удаляет все символы, кроме цифр и точки:

```php
private function sanitize_decimal_field( $field_name ) {
    if ( ! isset( $_POST[ $field_name ] ) ) {
        return '';
    }

    $value = wp_unslash( $_POST[ $field_name ] );
    $value = str_replace( ',', '.', $value );
    $value = preg_replace( '/[^0-9.]/', '', $value );

    if ( '' === $value ) {
        return '';
    }

    return (string) floatval( $value );
}
```

Метод `save_news_meta()` устроен аналогично и сохраняет только одно поле — кастомный URL для кнопки «Read More»:

```php
public function save_news_meta( $post_id ) {
    if ( ! $this->can_save_post( $post_id, 'my_catalog_news_meta_nonce', 'my_catalog_save_news_meta' ) ) {
        return;
    }

    update_post_meta( $post_id, self::NEWS_META_READ_MORE_URL,
        isset( $_POST['my_catalog_read_more_url'] )
            ? esc_url_raw( wp_unslash( $_POST['my_catalog_read_more_url'] ) )
            : '' );
}
```

### 3.4.2 Options API

Настройки отображаемых колонок таблицы товаров хранятся через стандартный Options API WordPress с использованием `register_setting()`. Опция зарегистрирована с типом `array` и callback'ом санитаризации:

```php
const OPTION_COLUMNS = 'my_catalog_product_table_columns';

public function register_settings() {
    register_setting(
        'my_catalog_settings',
        self::OPTION_COLUMNS,
        array(
            'type'              => 'array',
            'sanitize_callback' => array( $this, 'sanitize_columns' ),
            'default'           => $this->get_default_columns(),
        )
    );
}
```

Значение по умолчанию — пять колонок:

```php
private function get_default_columns() {
    return array( 'image', 'title', 'price', 'category', 'sku' );
}
```

Метод `sanitize_columns()` реализует подход whitelist (белого списка): каждый переданный ключ проходит `sanitize_key()`, после чего вычисляется пересечение с массивом доступных колонок. Если результат пуст — возвращаются колонки по умолчанию:

```php
public function sanitize_columns( $columns ) {
    $available = array_keys( $this->get_available_columns() );
    $columns   = is_array( $columns )
        ? array_map( 'sanitize_key', $columns )
        : array();
    $columns   = array_values( array_intersect( $columns, $available ) );

    return ! empty( $columns ) ? $columns : $this->get_default_columns();
}
```

Эта санитаризация гарантирует, что в базу данных попадут только валидные ключи колонок, а любые несуществующие или вредоносные значения будут отброшены.

### 3.4.3 REST API endpoint

Для обеспечения работы DataTables на фронтенде реализован endpoint `GET my-catalog/v1/product-table`. Маршрут регистрируется через `register_rest_route()` на хуке `rest_api_init`:

```php
public function register_rest_routes() {
    register_rest_route(
        'my-catalog/v1',
        '/product-table',
        array(
            'methods'             => WP_REST_Server::READABLE,
            'callback'            => array( $this, 'handle_rest_request' ),
            'permission_callback' => '__return_true',
        )
    );
}
```

Endpoint является публичным (`permission_callback => '__return_true'`) и принимает следующие параметры запроса:

- `search` — строка поиска;
- `category`, `tag` — фильтры по таксономиям;
- `price_min`, `price_max` — фильтр диапазона цены;
- `base_category`, `base_tag` — базовые фильтры из атрибутов шорткода;
- `start`, `length` — пагинация (DataTables protocol);
- `order_column`, `order_dir` — сортировка;
- `draw` — идентификатор запроса DataTables;
- `columns` — список запрашиваемых колонок.

Обработчик `handle_rest_request()` строит два `WP_Query`. Первый — для подсчёта общего количества записей (без фильтров и поиска), второй — с применением всех переданных фильтров, поиска, сортировки и пагинации:

```php
public function handle_rest_request( WP_REST_Request $request ) {
    // Парсинг и санитаризация параметров...
    $draw          = absint( $request->get_param( 'draw' ) );
    $search_term   = sanitize_text_field( (string) $request->get_param( 'search' ) );
    $length        = max( 1, absint( $request->get_param( 'length' ) ) );
    $start         = max( 0, absint( $request->get_param( 'start' ) ) );
    // ...

    // Запрос 1: общее количество записей
    $total_query = $this->run_products_query( array(
        'base_category'    => $base_category,
        'base_tag'         => $base_tag,
        'filter_category'  => '',
        'filter_tag'       => '',
        'filter_price_min' => null,
        'filter_price_max' => null,
        'search'           => '',
        'posts_per_page'   => 1,
        'offset'           => 0,
        'paginate'         => false,
    ) );

    // Запрос 2: с фильтрацией, поиском и пагинацией
    $filtered_query = $this->run_products_query( array(
        'base_category'    => $base_category,
        'base_tag'         => $base_tag,
        'filter_category'  => $filter_category,
        'filter_tag'       => $filter_tag,
        'filter_price_min' => $filter_price_min,
        'filter_price_max' => $filter_price_max,
        'search'           => $search_term,
        'posts_per_page'   => $length,
        'offset'           => $start,
        'paginate'         => true,
        'order_column'     => $order_column,
        'order_direction'  => $order_direction,
    ) );

    // Форматирование строк таблицы...
    return rest_ensure_response( array(
        'draw'            => $draw,
        'recordsTotal'    => absint( $total_query->found_posts ),
        'recordsFiltered' => absint( $filtered_query->found_posts ),
        'data'            => $data,
    ) );
}
```

Ответ следует протоколу DataTables: `recordsTotal` — общее количество записей без фильтров, `recordsFiltered` — количество после фильтрации, `data` — массив строк текущей страницы.

Метод `run_products_query()` собирает финальный `WP_Query`, комбинируя налоговые фильтры (`build_tax_query()`), фильтры по мета-полям (`build_meta_query()`), сортировку через маппинг колонок (`get_order_mapping()`) и кастомный поиск:

```php
private function run_products_query( $args ) {
    $query_args = array(
        'post_type'           => My_Catalog_Core::PRODUCT_POST_TYPE,
        'post_status'         => 'publish',
        'posts_per_page'      => ! empty( $args['paginate'] )
            ? (int) $args['posts_per_page'] : 1,
        'offset'              => ! empty( $args['paginate'] )
            ? (int) $args['offset'] : 0,
        'ignore_sticky_posts' => true,
        'no_found_rows'       => false,
    );

    // Налоговые фильтры (категория, тег)
    $tax_query = $this->build_tax_query(
        $args['base_category'], $args['base_tag'],
        $args['filter_category'], $args['filter_tag']
    );
    if ( ! empty( $tax_query ) ) {
        $query_args['tax_query'] = $tax_query;
    }

    // Фильтры по цене
    $meta_query = $this->build_meta_query(
        $args['filter_price_min'], $args['filter_price_max']
    );
    if ( ! empty( $meta_query ) ) {
        $query_args['meta_query'] = $meta_query;
    }

    // Кастомный поиск через posts_clauses
    if ( ! empty( $args['search'] ) ) {
        $query_args['my_catalog_search'] = sanitize_text_field( $args['search'] );
        add_filter( 'posts_clauses', array( $this, 'add_search_clauses' ), 10, 2 );
    }

    $query = new WP_Query( $query_args );

    if ( ! empty( $args['search'] ) ) {
        remove_filter( 'posts_clauses', array( $this, 'add_search_clauses' ), 10 );
    }

    return $query;
}
```

Поиск выполняется через фильтр `posts_clauses`. Метод `add_search_clauses()` добавляет `LEFT JOIN` к таблицам `wp_postmeta`, `wp_term_relationships`, `wp_term_taxonomy` и `wp_terms`, после чего формирует `WHERE` с `LIKE`-условиями по заголовку, содержанию, отрывку, мета-полям (цена, артикул, статус, вес) и названиям терминов таксономий `product_cat` и `product_tag`:

```php
public function add_search_clauses( $clauses, $query ) {
    global $wpdb;

    $search = $query->get( 'my_catalog_search' );
    if ( ! $search ) {
        return $clauses;
    }

    $like = '%' . $wpdb->esc_like( $search ) . '%';

    $clauses['join'] .= " LEFT JOIN {$wpdb->postmeta} AS my_catalog_meta_search
        ON ({$wpdb->posts}.ID = my_catalog_meta_search.post_id)";
    $clauses['join'] .= " LEFT JOIN {$wpdb->term_relationships} AS my_catalog_term_relationships
        ON ({$wpdb->posts}.ID = my_catalog_term_relationships.object_id)";
    $clauses['join'] .= " LEFT JOIN {$wpdb->term_taxonomy} AS my_catalog_term_taxonomy
        ON (my_catalog_term_relationships.term_taxonomy_id
            = my_catalog_term_taxonomy.term_taxonomy_id)";
    $clauses['join'] .= " LEFT JOIN {$wpdb->terms} AS my_catalog_terms
        ON (my_catalog_term_taxonomy.term_id = my_catalog_terms.term_id)";

    $clauses['where'] .= $wpdb->prepare(
        " AND (
            {$wpdb->posts}.post_title LIKE %1\$s
            OR {$wpdb->posts}.post_content LIKE %1\$s
            OR {$wpdb->posts}.post_excerpt LIKE %1\$s
            OR (
                my_catalog_meta_search.meta_key IN (%2\$s, %3\$s, %4\$s, %5\$s)
                AND my_catalog_meta_search.meta_value LIKE %1\$s
            )
            OR (
                my_catalog_term_taxonomy.taxonomy IN (%6\$s, %7\$s)
                AND my_catalog_terms.name LIKE %1\$s
            )
        )",
        $like,
        My_Catalog_Core::PRODUCT_META_PRICE,
        My_Catalog_Core::PRODUCT_META_SKU,
        My_Catalog_Core::PRODUCT_META_STOCK,
        My_Catalog_Core::PRODUCT_META_WEIGHT,
        My_Catalog_Core::PRODUCT_CATEGORY_TAXONOMY,
        My_Catalog_Core::PRODUCT_TAG_TAXONOMY
    );

    $clauses['groupby'] = "{$wpdb->posts}.ID";

    return $clauses;
}
```

Без `GROUP BY` множественные `LEFT JOIN` могли бы породить дубликаты строк (один товар с несколькими мета-полями или терминами), что привело бы к некорректному подсчёту `found_posts`. Группировка по `ID` устраняет эту проблему.

Маппинг колонок для сортировки определён в `get_order_mapping()`. Каждая колонка, поддерживающая сортировку, привязана к соответствующему полю `orderby`:

```php
private function get_order_mapping() {
    return array(
        'title'  => array( 'orderby' => 'title' ),
        'price'  => array(
            'meta_key' => My_Catalog_Core::PRODUCT_META_PRICE,
            'orderby'  => 'meta_value_num',
        ),
        'sku'    => array(
            'meta_key' => My_Catalog_Core::PRODUCT_META_SKU,
            'orderby'  => 'meta_value',
        ),
        'stock'  => array(
            'meta_key' => My_Catalog_Core::PRODUCT_META_STOCK,
            'orderby'  => 'meta_value',
        ),
        'weight' => array(
            'meta_key' => My_Catalog_Core::PRODUCT_META_WEIGHT,
            'orderby'  => 'meta_value_num',
        ),
    );
}
```

Связка пост-мета + Options API + кастомный REST endpoint образует трехуровневую систему хранения: мета-поля отвечают за контентные данные товаров и новостей, Options API — за пользовательские настройки отображения, REST API — за передачу этих данных на фронтенд в формате, совместимом с DataTables.
